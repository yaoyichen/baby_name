# 宝宝起名系统 — 产品需求文档 (PRD)

> 版本：v0.1 | 日期：2026-04-08 | 作者：Eason

---

## 1. 产品概述

### 1.1 产品定位

一个面向新生儿起名需求的**规则驱动 + AI 增强**的命名辅助工具。系统以穷举 + 多层过滤的方式，从数千个常用汉字中筛选出语音优美、五行相符、语义深远的候选名字，最终通过 LLM 精排输出高质量推荐榜单。

### 1.2 核心价值

| 维度 | 传统起名 | 本系统 |
|------|---------|--------|
| 覆盖度 | 依赖人工联想，遗漏多 | 枚举所有常用汉字组合，无遗漏 |
| 语音规则 | 靠直觉 | 精确的平仄、开口度算法校验 |
| 八字匹配 | 需专业知识 | 模块化八字/五行规则引擎 |
| 语义质量 | 主观判断 | LLM 多维度打分 + 解释 |
| 可复用性 | 一次性 | 任意姓氏、任意八字均可使用 |

### 1.3 当前阶段（v0.1）目标

为姚姓待出生宝宝起名，短期内作为**个人工具**使用，产出经语言学规则过滤的候选名单。

---

## 2. 用户画像

**主要用户**：即将迎来新生儿的父母，有文化素养，希望起一个有美感、有内涵且符合传统命理的名字。

**核心诉求**：
- 不遗漏好名字（穷举）
- 读音悦耳（语音规则）
- 字义积极、内涵深远（语义）
- 与生辰八字契合（命理）

## 3. 功能规划

### 3.1 版本路线图

```
v0.1 (当前)  → 汉字池构建 + 语言学过滤（平仄 + 开口度）
v0.2         → 八字五行过滤（用神、忌神、三才五格）
v0.3         → LLM 语义评分（批量打分 + Top-N 推荐）
v1.0         → Web 界面 / API 服务（产品化）
```

### 3.2 v0.1 详细功能

#### 3.2.1 汉字池构建
- **数据来源**：通过 GB2312 编码枚举，无需外部数据文件
- GB2312 收录 6763 个常用汉字（一级 3755 字 + 二级 3008 字），覆盖日常用字 99%+
- 自动过滤多音字（避免名字读音歧义）
- 预计算每字属性：拼音、声调、韵母、开口度

#### 3.2.2 语言学过滤
见第 4 章"过滤规则体系"

#### 3.2.3 输出
- 控制台打印：每阶段通过率统计 + 前 N 条样本预览
- CSV 文件：完整候选名单，含每个名字的属性标注

## 4. 过滤规则体系

### 4.1 第一层：语言学维度

#### 4.1.1 平仄规则（v0.1 实现）

中文四声分两类：
- **平声**：第一声（阴平）、第二声（阳平）
- **仄声**：第三声（上声）、第四声（去声）

**姚姓名字期望模式：平—仄—平**

| 位置 | 字 | 期望声调 | 说明 |
|------|----|---------|------|
| 1 | 姚 | 平（二声） | 姓氏固定，yáo 为阳平 ✓ |
| 2 | 名1 | 仄（三声或四声） | 过滤条件 |
| 3 | 名2 | 平（一声或二声） | 过滤条件 |

读音节奏效果示例：`姚(↗) 某(↓) 某(→/↗)`，形成"扬—抑—扬"的音律美感。

#### 4.1.2 开口度规则（v0.1 实现）

韵母按口腔开度分两类（对应传统音韵学的"四呼"）：

| 类型 | 韵母特征 | 四呼 | 示例 |
|------|---------|------|------|
| **大开口** | 韵母不以 i/u/ü 开头 | 开口呼 | a, o, e, ai, ao, an, ang, en, er 等 |
| **小开口** | 韵母以 i/u/ü 开头 | 齐齿呼/合口呼/撮口呼 | i, u, ü, in, un, ün, ia, ua 等 |

**过滤规则**：三字全部为小开口则拒绝（读起来像"咬着牙"说话）。

> 注：姚（yáo）含 "ao"，属大开口，故姚姓名字不会触发此规则。该规则对任意姓氏均有效，保留以确保框架完整可复用。

**可选严格模式**：要求名字后两字中至少一个为大开口（`require_open_in_name=True`）。

#### 4.1.3 未来语言学规则（v0.2+）

- **声母回避**：避免连续相同声母（如 f-f, zh-zh）
- **韵脚协调**：名字末字韵母与常用呼唤方式协调
- **字形均衡**：笔画数方差不过大，视觉上平衡（9±5 笔为佳）
- **偏旁多样**：三字不全用同一偏旁部首

### 4.2 第二层：八字五行维度（v0.2）

根据宝宝生辰八字分析：
- **日主**（日柱天干）强弱判断
- **用神**：补充日主所需五行
- **忌神**：需回避的五行

过滤逻辑：名字用字的五行属性需包含用神，不含忌神（或弱化忌神占比）。

数据需求：需要一份汉字→五行属性的映射表（基于康熙字典或字典偏旁推断）。

### 4.3 第三层：LLM 语义维度（v0.3）

对通过前两层的候选名字，调用 LLM API 进行多维度评分：

| 评分维度 | 权重 | 说明 |
|---------|------|------|
| 字义积极性 | 25% | 字的本义、引申义是否正面 |
| 意境组合感 | 25% | 两个名字字组合的意境是否深远 |
| 性别匹配度 | 20% | 字的气质是否符合性别预期 |
| 文学引用度 | 15% | 是否出自经典诗词/典故 |
| 独特性 | 15% | 避免过于大众化的名字 |

## 5. 技术架构

```
┌─────────────────────────────────────────────────────────┐
│                    main.py (CLI入口)                      │
│              python main.py --surname 姚                  │
└──────────────────────────┬──────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────┐
│                   generator.py (流水线)                   │
│   build_pool → cartesian_product → apply_filters → out   │
└────┬──────────────────────────────────────┬─────────────┘
     │                                      │
┌────▼──────────────┐           ┌───────────▼─────────────┐
│   char_pool.py    │           │   filters/ (规则层)      │
│ GB2312枚举6763字  │           │ ┌─ tone_filter.py       │
│ 过滤多音字        │           │ ├─ openness_filter.py   │
│ 预计算拼音属性    │           │ └─ (future filters...)  │
│ → 仄声池/平声池   │           └─────────────────────────┘
└───────────────────┘
```

### 5.1 字符池属性

每个汉字预计算以下属性（`CharAttr` dataclass）：

```python
@dataclass
class CharAttr:
    char: str        # 汉字
    pinyin: str      # 带声调数字的拼音，如 'yao2'
    tone: int        # 声调 1/2/3/4
    final: str       # 韵母，如 'ao', 'i', 'un'
    openness: str    # 'large'（大开口）或 'small'（小开口）
```

### 5.2 过滤器接口

```python
class NameFilter(ABC):
    @abstractmethod
    def check(self, chars: list[CharAttr]) -> bool:
        """返回 True 表示通过过滤，False 表示被拒绝"""
    
    @property
    @abstractmethod
    def description(self) -> str:
        """过滤器描述，用于统计输出"""
```

## 9. 后续产品化方向（v1.0+）

1. **Web 界面**：输入姓氏、八字、性别偏好，一键生成推荐榜单
2. **API 服务**：RESTful API，支持第三方集成
3. **多姓氏支持**：自动识别姓氏声调，调整平仄期望模式
4. **收藏 & 对比**：用户可收藏候选名字，进行对比评估
5. **分享功能**：生成名字解析卡片，方便与家人讨论
