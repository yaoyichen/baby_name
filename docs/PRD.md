# 择名台 — 产品需求文档 (PRD)

> 版本：v1.0 | 日期：2026-04-10 | 作者：Eason

---

## 1. 产品概述

### 1.1 产品名称

**择名台**（副标题：命理 · 古诗 · AI）

浏览器标签：`择名台 · 命理 · 古诗 · AI`

### 1.2 产品定位

一款面向新生儿起名的**多维命理 × 古典诗学 × AI 大模型**融合命名工作台。系统以汉字字库为基础，整合五行命理、三才五格数理、音韵学规则、古诗文典故，并借助 LLM 完成智能起名与专家级定制评分，实现"不遗漏好名字、不放过坏名字"的全流程命名体验。

### 1.3 核心价值

| 维度 | 传统起名 | 本系统 |
|------|---------|--------|
| 覆盖度 | 依赖人工联想，遗漏多 | 全量汉字枚举，零遗漏 |
| 语音规则 | 靠直觉 | 精确平仄/开口度/多音字算法 |
| 命理匹配 | 需专业知识 | 五行字库 + 三才五格自动计算 |
| 数理分析 | 手算易出错 | 81数理五档评分引擎（大吉/吉/平/凶/大凶） |
| 文学底蕴 | 主观联想 | 古诗文典故溯源（诗经/楚辞/唐诗/宋词等） |
| AI 起名 | 无 | LLM 批量起名（含BaZi用神约束） |
| AI 评名 | 无 | 多维度定制化打分（命理×AI联合评估） |
| 可复用性 | 一次性 | 任意姓氏/任意八字均可配置使用 |

### 1.4 当前阶段（v1.0）状态

系统已完整上线，核心功能全部可用，面向姚姓宝宝起名的个人工具，具备完整可视化 Web 界面。

## 2. 用户画像

**主要用户**：即将迎来新生儿的父母，有文化素养，希望起一个美感、内涵、命理兼备的名字。

**核心诉求**：
- 不遗漏好名字（穷举候选）
- 读音悦耳（音韵规则）
- 字义积极、意境深远（语义/文学）
- 与生辰八字契合（命理/五行）
- 数理格局稳固（三才五格）
- 能借助 AI 快速筛选（效率）

## 3. 功能模块

### 3.1 浏览模块（Browse）

**字库与候选集：**
- 数据来源：`data/all_chars.json`（核心字库）+ `data/lookup_chars.json`（扩展查阅库）
- 核心字库以 `data/full_wuxing_dict.py` 为五行/笔画权威来源，与 `data/chinese_chars.csv` 取交集
- 支持 `data/yijing_lucky.json`（易经50吉字）优先排序

**筛选维度（左侧边栏）：**

| 筛选项 | 说明 |
|--------|------|
| 姓氏 | 支持1-2字，影响三才五格计算 |
| 包含多音字 | 开关，默认开启 |
| 名一/名二 五行 | 多选（木火土金水） |
| 名一/名二 声调 | 1/2/3/4声，支持多选 |
| 名一/名二 开口度 | 大开口/小开口，含hover释义 |
| 黑名单 | 屏蔽指定字，持久化到服务端 `blacklist.json` |
| 排序 | 易经吉字优先 / 笔画正序 / 笔画倒序 |

**候选字池面板（可折叠）：**
- 展示当前过滤后的名一/名二字池，含五行色块、拼音、笔画数
- 字符旁支持一键屏蔽

**名字卡片：**
- 展示：姓名 + 拼音 + 五行/声调/笔画/开口度/多音/易经角标 + 古诗文来源
- 右上角显示三才五格综合评分徽章（颜色分5档）
- 收藏按钮（♡/♥），支持跨 tab 持久化

**分页：** 每页 20 条，支持翻页导航

---

### 3.2 收藏夹模块（Favorites）

- 所有收藏的名字（含单字名/双字名）按时间倒序列出
- 顶部工具栏：
  - 排序下拉（默认 / 总分 / 总格 / 人格）
  - 搜索框（支持汉字名或拼音模糊匹配）
  - 清空收藏按钮
- 每张卡片底部展示三才五格综合评分 pill（含等级标签）
- **Hover 五格弹窗**：鼠标悬停评分 pill 时，弹出完整五格三才数理分析浮层

---

### 3.3 查名模块（Lookup）

- 输入姓+名（1-3字），查询字库内所有字的详细属性
- 优先查 `all_chars.json`，其次 `lookup_chars.json`，未收录字显示"仅查阅"标注
- 展示五格三才完整分析（支持单字名假添一数规则）

---

### 3.4 AI 起名模块（AI Naming）

**输入：**
- 模型选择（支持多厂商大模型，见第5章）
- 生成数量（10/20/50/100）
- 姓氏
- 宝宝八字/五行信息（可自由编辑文本框，持久化到 localStorage）

**处理：**
- 构建 Prompt：包含八字背景 + 当前字池（名一/名二各字）约束
- 通过本地 `server.py` SSE 代理转发 LLM 流式请求
- LLM 输出 JSON 数组，前端逐步解析渲染

**输出：**
- 名字表格：序号 / 姓名 / 拼音 / 名一属性 / 名二属性 / AI评分 / 原因 / 收藏
- 支持收藏（字库外字自动生成兜底属性）
- 按 AI 评分倒序展示

---

### 3.5 AI 评名模块（AI Evaluation）

**输入：**
- 模型选择
- 命主背景 & 评分体系（可编辑文本框，包含八字、用神忌神、家族期许、各维度评分标准）
- 最多5个候选名字

**评分体系（总分100分）：**

| 维度 | 满分 | 评分方式 | 说明 |
|------|------|----------|------|
| 八字五行契合度 | 45分 | AI | 字义/字根五行是否契合用神，含一票否决红线 |
| 三才五格数理 | 15分 | **系统自动计算** | 复用系统五格引擎，0-100映射至0-15 |
| 精神内核意境 | 25分 | AI | 双字单意、领袖气质、意境深度 |
| 音律字型美学 | 15分 | AI | 平仄起伏、字型结构错落 |

**输出（结构化评分卡片）：**
- 每个名字一张卡片，含：大字展示 + 圆形总分 + 评级徽章 + 四维度进度条（AI/系统标签区分）+ 专家深度点评 + 最终判定
- 触碰红线（一票否决）时显示红色警示横幅
- 按总分倒序排列
- 流式响应过程中显示 loading 动画 + 字符接收进度

## 4. 命理引擎（核心算法）

### 4.1 五行字库

- 权威来源：`data/full_wuxing_dict.py`（含7000+字的康熙笔画数 + 五行属性）
- 拼音/声调/开口度来源：`data/chinese_chars.csv`
- 多音字单独处理，保留所有读音供前端展示

### 4.2 三才五格数理

详见 `docs/wuge_sancai.md`，核心要点：

**双字名格位公式：**
| 格位 | 公式 | 权重 |
|------|------|------|
| 天格 | 姓+1 | 5% |
| 人格 | 姓+名一 | 45% |
| 地格 | 名一+名二 | 25% |
| 总格 | 姓+名一+名二 | 20% |
| 外格 | 名二+1 | 5% |

**单字名：** 启用假添一数规则（地格=名一+1，总格=人格，外格=2）

**81数理五档评分：** 大吉(100)/吉(80)/平(60)/凶(40)/大凶(20)

**三才五行修正：**
- 相生：`score += (100 - score) × 0.3`
- 相克：`score -= (score - 20) × 0.2`

**连凶惩罚：** 核心三格（人/地/总）中≥2个大凶，最终分×0.9

**评分等级：**
| 分数 | 等级 | 颜色 |
|------|------|------|
| 90-100 | 上吉 | 翠绿 `#2E7D32` |
| 80-89 | 中吉 | 湖蓝 `#0277BD` |
| 65-79 | 平顺 | 琥珀 `#F57C00` |
| 50-64 | 偏凶 | 砖红 `#C62828` |
| 0-49 | 大凶 | 深暗红 `#b71c1c` |

### 4.3 易经吉字

来源：`data/yijing_lucky.json`，收录易经最吉利50字，浏览时优先排序并显示「易经」角标。

## 5. 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                  analysis/index.html (前端单页应用)            │
│  浏览 | 收藏夹 | 查名 | AI起名 | AI评名                        │
└──────────────────────────┬──────────────────────────────────┘
                           │ HTTP (本地)
┌──────────────────────────▼──────────────────────────────────┐
│                       server.py (本地Python服务)              │
│  GET  /analysis/       静态文件服务                            │
│  GET  /data/*.json     数据文件服务                            │
│  POST /api/blacklist   黑名单持久化 (blacklist.json)           │
│  POST /api/ai-names    LLM SSE 流式代理                       │
└──────────────────────────┬──────────────────────────────────┘
                           │ HTTP (apikey header)
┌──────────────────────────▼──────────────────────────────────┐
│              LLM API (openai.infly.tech/v1)                   │
│  支持：DeepSeek / Qwen / GLM / Llama / Mistral 等             │
└─────────────────────────────────────────────────────────────┘

数据文件：
  data/full_wuxing_dict.py  → 五行/笔画权威字库（7000+字）
  data/chinese_chars.csv    → 拼音/声调/开口度属性
  data/all_chars.json       → 生成的核心候选字库
  data/lookup_chars.json    → 扩展查阅字库（仅查名）
  data/blacklist.json       → 用户黑名单（持久化）
  data/yijing_lucky.json    → 易经50吉字
  data/poems/               → 古诗文数据（诗经/楚辞/唐诗等）

生成脚本：
  src/generate_all_chars.py   → 生成 all_chars.json
  src/generate_lookup.py      → 生成 lookup_chars.json
  src/char_pool.py            → 字库核心逻辑
```

### 5.1 前端技术栈

- 纯原生 HTML + CSS + JavaScript（无框架依赖）
- CSS 变量支持日间/浅色模式
- localStorage 持久化用户配置（收藏夹、AI配置、黑名单缓存）
- Fetch API + ReadableStream 实现 SSE 流式渲染

### 5.2 服务端

- Python 3 标准库 `http.server`
- 单文件 `server.py`，无第三方依赖
- LLM 代理使用 `urllib.request`，支持 SSE 透传

## 7. 非功能需求

| 需求 | 目标 |
|------|------|
| 性能 | 页面初始化（含 all_chars.json 加载）< 2秒 |
| 响应性 | 滚动/筛选操作无卡顿，分页每次渲染 < 100ms |
| 可靠性 | LLM 请求失败时前端优雅降级，显示错误信息 |
| 可维护性 | 五格算法、字库、提示词均可独立修改 |
| 本地优先 | 无需联网即可使用浏览/收藏/查名/五格功能 |

## 8. 启动说明

```bash
# 安装依赖（仅生成脚本需要）
pip install -r requirements.txt

# 重新生成字库（可选，full_wuxing_dict.py 更新后执行）
python src/generate_all_chars.py
python src/generate_lookup.py

# 启动本地服务
python server.py
# 访问 http://localhost:8080/analysis/
```
